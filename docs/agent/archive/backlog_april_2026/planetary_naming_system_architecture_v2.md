## Three Tiers of Naming (Never Mix These)

### Tier 1 — Scientific Catalog (working correctly, do not change)
- Machine-generated identifiers for all systems during exploration
- Format: `AOL-732356` (system), `AOL-732356-b`, `AOL-732356-c` (bodies)
- Generated by `NameGeneratorService#generate_identifier`
- **Persists until first Tier 1 habitation module lands** — this is the
  naming trigger (Phase 4 of implementation phases)
- Never replaced at survey, discovery, or any earlier point

### Tier 2 — Known/Historic Systems (Sol, Alpha Centauri etc)
- Real established names present from game start
- Loaded from JSON system data files
- Planets get positional names derived from system name (see conventions below)
- Cultural/historic names stored as **aliases** in `properties` JSONB field
  - Canonical: `Sol I` → Alias: `Mercury`
  - Canonical: `Sol III` → Alias: `Earth`
- Alias storage already implemented — `properties['aliases']` array

### Tier 3 — Settlement/Proper Names (what PlanetNameService handles)
- Assigned only when **first Tier 1 habitation module lands**
- System gets a proper name at this point, replacing the catalog identifier
- All world names in the system are derived from the system name
- System name drawn from `config/names.yml` (sci-fi universe categories:
  Star Trek, Star Wars, Babylon 5, Battlestar Galactica, Dune, plus generic)

## Naming Conventions

### Star Naming
- Primary star: `[System Name]-A` (e.g. `Eden-A`)
- Binary/trinary: `[System Name]-B`, `[System Name]-C`

### Planet Naming — Convention is System-Level
**The naming convention is set once per system and all bodies follow it.
Never mix conventions within a system.**

| Convention | Primary World | Secondary Worlds | Moons | Used For |
|---|---|---|---|---|
| Roman | `Eden Prime` | `Eden I`, `Eden II`, `Eden III` | `Eden Prime I`, `Eden II I` | Terrestrial, habitable, general, desert, volcanic, military |
| Greek | `Aqua Alpha` | `Aqua Beta`, `Aqua Gamma`, `Aqua Delta` | `Aqua Alpha I`, `Aqua Beta I` | Oceanic, rocky, research, scientific flavor |
| Numeric | `Forge-1` | `Forge-2`, `Forge-3` | `Forge-1-I` | Industrial, mining |
| Catalog | `AOL-732356` | `AOL-732356-b`, `AOL-732356-c` | — | Unsettled procedural |

### Primary World Rule
Every settled system has exactly one primary world — the most habitable or
main terraforming target:
- Roman convention → `[System] Prime`
- Greek convention → `[System] Alpha`

`Prime` and `Alpha` are equivalent in meaning. Convention determines which
is used. **Never use both in the same system.**

### Moon Naming
Always Roman numerals regardless of system convention:
- `[Planet Name] [Roman Numeral]`
- Examples: `Eden Prime I`, `Eden Prime II`, `Eden II I`, `Aqua Alpha I`

### Greek Letter Order (when used)
Always follow proper alphabetical sequence:
Alpha, Beta, Gamma, Delta, Epsilon, Zeta, Eta, Theta, Iota, Kappa,
Lambda, Mu, Nu, Xi, Omicron, Pi, Rho, Sigma, Tau, Upsilon, Phi, Chi,
Psi, Omega

## Uniqueness Rules

### System Name Uniqueness
**System names must be globally unique across the entire game.**
Since all world names derive from the system name, enforcing uniqueness
at system level guarantees all world names are unique automatically.

The current `unique?` check in `PlanetNameService` checks at the wrong
level (planet names). It must check **system name** uniqueness:

```ruby
def unique_system_name?(name)
  return false if @used_names.include?(name)
  !SolarSystem.exists?(name: name)
end
```

### System Name Disambiguation
When a preferred system name is already taken, try in order:
1. Next name from the category pool
2. Add a prefix: `New Eden`, `Greater Eden`, `Lesser Eden`
3. Add a positional suffix from the suffixes pool: `Eden Minor`, `Eden II`

The suffix/prefix is part of the **system name** — so `New Eden`'s primary
world is `New Eden Prime`, not `Eden Prime`.

### Star Name Scoped Uniqueness
Star names (e.g. `Sol`) are unique within their solar system only.
`Sol` can exist in multiple galaxies — this is intentional per
`star_naming_architecture.md`.

## Aliases

Cultural, historical, or player-assigned names are stored as aliases:

```json
{
  "name": "Sol I",
  "properties": {
    "aliases": ["Mercury"]
  }
}
```

- Aliases do not replace canonical positional names
- All aliases map to the primary body ID
- Already implemented in `SystemBuilderService` via properties merging
- Do not change the alias storage mechanism

## Required Code Changes

### 1. Clean up `planet_names.json`
Remove all embedded suffixes from name strings. Pool names must be
**pure base system names only** — no `Alpha`, `Beta`, `Prime`, `I`, `II`,
`Major`, `Minor` etc hardcoded into them.

**Wrong (current):**
```json
"rocky": ["Stone Prime", "Boulder Alpha", "Crag Beta", "Ridge Delta"]
```

**Correct:**
```json
"rocky": ["Stone", "Boulder", "Crag", "Ridge", "Peak", "Canyon", "Plateau",
          "Mesa", "Butte", "Spire", "Summit", "Gorge", "Ravine", "Cliff"]
```

The `suffixes` array remains for system name disambiguation only.

### 2. Rewrite `PlanetNameService`

Key changes:
- `generate_system_name` returns system name + convention as a pair
- `unique?` checks system name against `SolarSystem` table, not planet names
- `add_suffix_to_name` is deterministic using system_identifier as seed
- Add `determine_convention(category)` method
- Add `generate_world_name(system_name, convention, position)` method
- Add `generate_moon_name(planet_name, position)` method

```ruby
def determine_convention(category)
  case category
  when 'oceanic', 'ice', 'rocky', 'research', 'icy_moon'
    :greek
  when 'industrial', 'mining'
    :numeric
  else
    :roman
  end
end

def generate_world_name(system_name, convention, position)
  case convention
  when :roman
    position == 0 ? "#{system_name} Prime" : "#{system_name} #{roman_numeral(position)}"
  when :greek
    position == 0 ? "#{system_name} Alpha" : "#{system_name} #{greek_letter(position)}"
  when :numeric
    "#{system_name}-#{position + 1}"
  end
end

def generate_moon_name(planet_name, position)
  "#{planet_name} #{roman_numeral(position + 1)}"
end

def add_suffix_to_name(base_name, system_identifier)
  suffixes = @names_data['suffixes'] || ['I', 'II', 'III']
  seed = system_identifier.to_s.hash.abs % suffixes.length
  "#{base_name} #{suffixes[seed]}"
end
```
