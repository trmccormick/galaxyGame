# STARSIM_GENERATION_RULES.md

## Procedural Generation Constraints

### Rule 1: Anchor Priority
The generator should favor systems with planets $> 95\ M_{\oplus}$ (Saturn-mass) for stable, long-term wormhole endpoints. High-mass planets provide gravitational stability and reduce maintenance costs for wormhole links.

### Rule 2: The 5 AU Buffer
Natural wormholes generated by StarSim should not spawn within the "Habitable Zone" of a G-type star (typically inside 5 AU). This prevents tidal instability and protects habitable worlds from catastrophic gravitational effects.

### Rule 3: Connectivity Cap
A system must not exceed `MAX_WORMHOLES_PER_SYSTEM` to prevent "Hub Congestion"—where one system becomes an inescapable transit point. This cap ensures a balanced, navigable network and avoids single-point bottlenecks.

---

These rules must be enforced by all procedural generation jobs and referenced by any code responsible for wormhole placement or network topology.
